0. 운영 원칙

GJU-reserve는 학생 예약을 빠르게 받되, 학과 장비와 공간의 실제 운영 기록을 남기는 것이 목적입니다.

시스템 기록 우선

예약, 승인, 반납, 보고서, 특강 신청은 Admin 화면의 기록을 기준으로 처리합니다. 구두로 처리한 예외도 공지, Slack, Admin 상태 중 하나로 남깁니다.

개인정보 최소 공유

연락처와 학번은 예약 처리에 필요한 경우에만 사용합니다. 화면 캡처를 공유할 때는 비밀번호, Webhook, FTP 정보가 보이지 않게 합니다.

삭제보다 비활성화

학생 계정 삭제는 신중히 처리하고, 장비는 가능한 기자재 제거로 숨깁니다. 과거 예약 내역을 추적할 수 있어야 합니다.

운영 전 점검

학기 시작 전에는 수업 차단 일정, 기자재 상태, 출력비 계좌, 특강 모집 상태, 조교 계정 비밀번호를 확인합니다.

1. 매일 운영 순서

학기 중에는 아래 순서대로 보면 대부분의 업무가 처리됩니다.

가입 승인신규 학생 확인
기자재 승인대여 가능 여부 확인
오늘 예약대여, 반납, 사용 완료
보고서/공지스튜디오 보고서, 공지 갱신

2. 계정과 권한

학생은 직접 가입하고, 조교가 승인해야 예약할 수 있습니다. 인증 절차는 학과 내부 링크 공유와 실명 가입을 전제로 합니다.

구분할 수 있는 일주의
학생 공지 열람, 특강 신청, 예약 신청, 내 예약 확인, 스튜디오 보고서 제출, 내 정보 수정 승인 전에는 예약 불가. 이름, 학번, 신분, 연락처를 실제 정보로 유지해야 합니다.
조교/Admin 학생 승인, 예약 처리, 장비 등록/기자재 제거, 보고서 확인, 특강/공지/설정 관리 비밀번호와 Webhook은 문서나 채팅에 남기지 않습니다. 조교 교체 시 계정 비밀번호를 반드시 변경합니다.
중요

공용 PC에서 Admin을 사용한 뒤에는 반드시 로그아웃합니다. 학생 화면과 Admin 화면이 같은 앱 안에 있으므로 계정 역할에 따라 보이는 메뉴가 달라집니다.

3. 대시보드

대시보드는 조교가 하루 업무를 시작할 때 보는 첫 화면입니다. 카드 클릭 시 해당 관리 페이지로 이동합니다.

가입 승인 대기

새로 가입한 학생 수입니다. 클릭하면 학생 승인으로 이동합니다.

기자재 승인 대기

조교 승인 전 기자재 예약입니다. 장비 수량과 일정 충돌을 확인합니다.

오늘 예약

오늘 날짜의 기자재, 스튜디오, 암실, 출력 예약입니다.

보고서 확인 필요

스튜디오 사용 후 보고서가 필요하거나 확인해야 할 항목입니다.

4. 학생 승인

학생 가입 신청을 승인, 반려, 대여금지 처리할 수 있습니다. 상단 필터로 전체, 승인대기, 승인완료, 반려, 대여금지 상태를 나눠 보고, 이름, 학번, 신분, 상태 기준으로 정렬할 수 있습니다.

버튼의미언제 사용하나
승인학생 예약 가능학과 학생 또는 허용 대상임을 확인했을 때
반려예약 불가정보가 틀렸거나 학과 대상자가 아닐 때
대여금지기간 동안 예약 불가규칙 위반 또는 관리자 판단으로 일정 기간 막아야 할 때. 경고 메모 자체는 상태를 바꾸지 않습니다.
비번 리셋임시 또는 지정 비밀번호 발급학생이 비밀번호를 잊었을 때
경고 메모관리자용 기록 저장학생 상태 변경 없이 사유를 남겨 조교끼리 공유해야 할 때
대여금지 기간

대여금지는 학생 행의 드롭다운에서 1주일, 2주일, 1달, 1학기 중 선택하면 즉시 적용됩니다. 대여금지 중인 학생은 공지는 볼 수 있지만 예약은 할 수 없습니다.

상태 필터와 정렬

새 가입자는 승인대기에서 먼저 확인합니다. 처리 결과를 점검할 때는 승인완료, 반려, 대여금지 필터를 사용하고, 학번순 확인이 필요하면 테이블의 학번 헤더를 누릅니다.

5. 예약 관리

예약 관리는 전체, 기자재, 암실, 스튜디오, 출력 탭으로 나뉩니다. 예약 카드에는 학생명, 날짜, 연락처, 상세 항목, 상태 변경 버튼이 표시됩니다.

예약 종류확정 방식조교가 해야 할 일주요 상태
기자재 조교 승인 선택 장비, 대여일, 대여/반납 시간, 2박3일 여부, 장비 상태 확인 후 승인합니다. 승인 대기 → 승인 → 대여 완료 → 반납 완료
스튜디오 자동 확정 공간과 연속 타임을 확인하고, 사용 후 보고서 제출 여부를 확인합니다. 자동 확정 → 사용 완료
암실 자동 확정 정원, 야간/주말 연락 여부, 약품 사용량 입력 상태를 확인합니다. 자동 확정 → 사용 완료
출력실 자동 확정 사용 시간이 10:00-19:00 범위인지, 파일 준비/입금/정리 규칙을 안내합니다. 자동 확정 → 사용 완료
취소 처리 기준

학생이 연락 없이 오지 않았거나, 파일/장비 준비가 규정과 맞지 않거나, 수업/공휴일/현장 상황으로 사용이 불가하면 관리자 취소로 정리하고 필요하면 대여금지를 적용하거나 경고 메모를 남깁니다.

기자재 상태 필터

기자재 탭에서는 예약 종류 탭 아래에 전체, 대여완료, 반납완료 필터가 표시됩니다. 대여 완료는 아직 학생이 장비를 가지고 있는 상태이고, 반납 완료는 장비 회수가 끝난 상태입니다.

6. 기자재 관리

카테고리, 관리처, CSV 업로드, 등록된 전체 기자재 탭을 관리합니다. 장비는 가능한 개체별 코드로 관리하는 것이 좋습니다.

1

장비 추가

장비명, 카테고리, 수량, 코드 prefix, 관리처, 비고를 입력합니다. 수량이 3이면 코드가 붙은 개체 3개로 관리됩니다.

2

카테고리 추가

Body, Lens, Lighting 외에 Tripod, Monitor 같은 분류가 필요하면 먼저 카테고리를 추가합니다.

3

CSV 업로드

엑셀에서 CSV로 저장한 뒤 내용을 붙여넣고 미리보기를 눌러 행이 제대로 읽혔는지 확인합니다.

4

기자재 제거

수리 중, 분실, 운영 제외 장비는 기자재 제거 버튼으로 목록에서 숨깁니다. 과거 예약 내역 보존을 위해 데이터는 삭제하지 않습니다.

관리처운영 방식
극기관기본 예약 가능 장비입니다. 학생이 직접 선택하고 조교가 승인합니다.
판타지랩기본값은 문의 전용입니다. 판타지랩 조교에게 연락 후 대여하도록 안내합니다.

7. 스튜디오 보고서

학생은 스튜디오 사용 후 서비스 안에서 보고서를 제출합니다. Admin의 보고서 화면에는 제출된 보고서가 누적됩니다.

예약 확정학생이 연속 최대 3타임 예약
사용 완료정리정돈 확인
보고서 제출사용 후 48시간 이내
Admin 확인파손/이상 여부 확인
보고서 미제출

사용 후 48시간이 지나도 보고서가 없으면 학생에게 연락합니다. 반복 미제출이나 부적절 사용은 경고 메모로 기록하고, 실제 예약을 막아야 하면 대여금지를 별도로 적용합니다.

검색과 정렬

보고서 화면에서는 이름, 학번, 날짜, 스튜디오, 예약ID로 검색할 수 있습니다. 제출일, 사용일, 이름, 파손 여부 정렬을 사용하면 학기 말 정산이나 파손 확인이 쉬워집니다.

8. 비교과 특강

Admin에서 특강을 등록하면 학생 화면과 홈의 모집중 특강 카드에 표시됩니다.

입력 필드설명
특강명, 특강일, 시간, 장소학생 리스트와 CSV에 표시되는 기본 정보입니다.
강사명, 강사 소속, 담당교수특강 안내와 내부 기록에 사용합니다.
대상 학년, 모집인원, 신청인원학생이 신청 가능 여부를 판단할 때 참고합니다.
특강 내용, 비고학생 안내와 운영 메모입니다.
진행상태모집중, 진행완료, 취소 중 선택합니다. 모집중인 특강만 학생 신청 버튼이 표시됩니다.
CSV 내보내기

특강 결과는 Admin의 CSV 내보내기로 엑셀에서 열 수 있습니다. 신청자 이름, 학번, 신분, 연락처, 신청일이 포함됩니다.

수정 화면

특강 리스트에서 수정을 누르면 등록 폼 대신 수정 폼만 표시됩니다. 수정을 저장하거나 닫은 뒤 새 특강을 등록합니다.

9. 공지사항

공지사항은 학생 홈 최상단과 공지 탭에 표시됩니다. 학생은 공지를 누르면 바텀시트로 전체 내용을 봅니다.

항목운영 기준
카테고리일반, 비교과, 특강, 장비/시설, 긴급 중 선택합니다.
상단 고정중요 공지나 일정 변경은 고정합니다. 고정 공지는 홈에서 먼저 보입니다.
신청/참고 링크Slack 신청 링크, 외부 폼, 참고 문서가 있으면 입력합니다.
짧게 쓰기

홈에는 본문이 짧게 보이므로 제목에 핵심을 넣고, 자세한 규정은 본문에 정리합니다.

10. 설정과 차단 일정

설정은 예약 가능 규칙에 직접 영향을 줍니다. 학기 초 또는 방학 전후에 반드시 확인합니다.

운영 설정

  • 출력비 계좌 안내
  • 암실 최대 인원
  • 출력실 시작/종료 시간

수업/학기 차단 일정

  • 시설: 스튜디오, 암실, 기자재, 출력실
  • 요일, 시작일, 종료일, 시간
  • 공간/메모: Studio A/B, 수업명 등
학기 시작 전 확인

수업 때문에 사용 불가한 요일과 시간이 생기면 차단 일정에 등록합니다. 학생 예약 캘린더에는 차단 마커가 표시됩니다.

11. 학생 화면 안내

학생 문의가 들어오면 아래 구조로 안내하면 됩니다.

탭학생이 하는 일문의 대응
홈공지, 모집중 특강, 다음 예약 확인중요 안내는 공지로 올렸는지 확인합니다.
예약기자재, 스튜디오, 출력실, 암실 예약승인 대기/대여금지 상태면 예약이 막힙니다.
내 예약본인 예약 확인/취소취소 가능 여부와 조교 연락 필요 여부를 안내합니다.
보고서스튜디오 사용 후 보고서 제출사용일 이후 보고서 작성 항목이 뜹니다.
특강모집중 특강 신청진행완료/취소 상태는 신청 버튼이 없습니다.
공지전체 공지 확인공지 클릭 시 바텀시트로 자세히 표시됩니다.
마이이름/연락처 수정, 비밀번호 변경, 로그아웃학생 이메일 수정은 받지 않습니다. 연락처 최신화가 중요합니다.

12. Slack 알림

예약과 주요 이벤트는 Slack 채널 #예약_현황automatic에 텍스트로 공유됩니다.

주로 올라오는 내용

  • 학생 가입 신청
  • 기자재 예약 승인 요청
  • 스튜디오/암실/출력 예약 확정
  • 스튜디오 보고서 제출

주의

  • Webhook URL은 서버 환경변수로만 관리합니다.
  • 전화번호 등 개인정보는 필요한 범위에서만 공유합니다.
  • Slack 전송 실패가 있어도 Admin 예약 내역을 기준으로 처리합니다.

13. 로그/세션

로그/세션 화면은 계정 접속, 예약 처리, 학생 상태 변경 같은 운영 기록을 확인하는 곳입니다. 삭제보다 확인과 원격 로그아웃을 우선합니다.

영역정렬/필터언제 쓰나
현재 로그인 세션최근 로그인방금 접속한 학생 또는 조교 계정을 확인할 때 기본으로 봅니다.
현재 로그인 세션만료 임박공용 PC, 오래된 접속, 의심 세션을 정리할 때 사용합니다.
현재 로그인 세션사용자명특정 학생이나 조교의 접속 기기를 찾을 때 사용합니다.
활동 로그작업 분류 필터로그인, 예약, 학생, 기자재, 특강, 보고서, 공지, 설정 중 필요한 기록만 좁혀 봅니다.
활동 로그최신순/오래된순일반 확인은 최신순, 문제가 발생한 과정을 시간대로 따라갈 때는 오래된순을 사용합니다.
의심 세션 처리

낯선 IP나 기기가 보이면 먼저 학생 또는 조교에게 확인하고, 필요한 경우 해당 세션을 원격 로그아웃합니다. 비밀번호가 공유됐다고 판단되면 계정 비밀번호도 변경합니다.

14. 배포와 백업

조교가 직접 배포할 일은 많지 않지만, 운영자가 알아야 할 구조와 백업 원칙은 아래와 같습니다.

서비스 구조

  • Dothome: 학생과 Admin이 보는 정적 화면을 제공합니다.
  • Cloudflare Worker: API와 운영 DB 역할을 합니다.
  • Dothome의 /api/* 요청은 PHP 프록시를 통해 Worker로 전달됩니다.

배포 확인

  • 화면 파일이 바뀌면 Dothome에 index.html, app.js, styles.css, js/*, api/*를 업로드합니다.
  • 브라우저 캐시 때문에 버전 문자열이 바뀌어야 새 JS가 확실히 로드됩니다.
  • 업로드 후 학생 서비스 주소에서 새 화면이 보이는지 확인합니다.

백업 JSON

  • Admin 설정 화면의 백업 JSON 버튼으로 운영 데이터를 내려받을 수 있습니다.
  • 백업에는 사용자, 예약, 보고서, 특강, 공지, 로그가 포함될 수 있으므로 외부에 공유하지 않습니다.
  • 학기 종료, 조교 교체, 큰 장비 정리 전에는 백업을 남깁니다.

민감정보

  • FTP 비밀번호, Admin 비밀번호, Slack Webhook은 문서에 적지 않습니다.
  • 일반 FTP는 암호화되지 않으므로 사용 후 비밀번호 변경을 권장합니다.
  • 운영 DB 초기화 기능은 없습니다. 실수로 전체 데이터를 지우는 방식의 운영은 금지합니다.

15. 문제 발생 시

상황먼저 확인할 것처리
학생이 로그인이 안 된다고 함 아이디가 이메일 또는 학번인지, 승인 상태인지 확인 Admin 학생 승인에서 상태 확인 후 필요하면 비밀번호 리셋
예약 버튼이 안 보임 학생 승인 상태, 대여금지 상태, 특강 진행상태 확인 승인 또는 제한 해제 후 다시 접속 안내
기자재 수량이 맞지 않음 Admin 기자재 관리에서 기자재 제거/문의전용 상태 확인 장비 추가 또는 기자재 제거 처리, 필요하면 CSV로 일괄 반영
Slack에 알림이 안 올라옴 Admin 예약 내역에는 저장됐는지 확인 예약 내역을 우선 기준으로 처리하고, 개발 담당자에게 Webhook 상태 확인 요청
페이지가 비어 보임 브라우저 새로고침, 다른 브라우저, 네트워크 상태 확인 지속되면 화면 캡처와 발생 시간을 기록해 개발 담당자에게 전달
개발자는 수정됐다고 하는데 실제 사이트는 그대로임 Dothome 정적 파일이 최신인지, index.html의 버전 문자열이 바뀌었는지 확인 Dothome 업로드 후 새로고침하고, 필요하면 브라우저 캐시를 지웁니다.
보고서나 예약을 찾기 어려움 검색어가 이름, 학번, 날짜, 스튜디오, 예약ID 중 하나인지 확인 예약 관리 검색 또는 보고서 검색/정렬을 사용하고, 기자재는 대여완료/반납완료 필터로 좁힙니다.
누가 상태를 바꿨는지 확인해야 함 로그/세션의 활동 로그에서 학생, 예약, 기자재 등 작업 분류를 선택 최신순으로 최근 변경을 보고, 흐름을 따라가야 하면 오래된순으로 전환합니다.

# 스마트 예약 운영 고도화 기능 기획서

작성일: 2026-07-20

상태: 검토용 시뮬레이션 설계

대상: GJU-reserve 학생 앱과 관리자 대시보드

## 1. 목표

GJU-reserve의 현재 예약·승인·대여·반납 흐름을 유지하면서, 학생이 예약을 더 쉽게 다시 신청하고 관리자가 운영 병목을 빠르게 발견하도록 만든다.

이 설계는 기능을 많이 늘리는 대신 기존 화면의 역할을 강화한다.

- 학생 홈: 최근 예약, 그룹형 즐겨찾는 장비, 다시 예약, 꼭 필요한 알림
- 관리자 대시보드: 혼잡 시간, 장비 가동률, 취소율, 수요·반납 위험
- 예약 화면: 예약 불가 상황에서만 날짜·시간·대체 장비 추천

## 2. 현재 프로젝트를 기준으로 한 출발점

프로젝트에는 이미 다음 기반이 있다.

- 학생은 기자재·스튜디오·암실·출력실 예약을 신청할 수 있다.
- 기자재는 `승인 대기 → 승인 완료 → 대여 중 → 반납 완료` 상태로 관리한다.
- 관리자 대시보드는 승인·대여·반납·보고서 처리 큐, 이번 주 예약 수, 기자재 가용률, 수리중 기자재, 취소/반려 수를 보여 준다.
- 예약 데이터에는 날짜, 시간, 장비 ID, 시설, 예약 상태가 남아 있어 운영 통계를 계산할 수 있다.
- 모바일 앱은 네이티브 예약 알림을 예약·동기화할 수 있다.

따라서 1차 기능은 새 분석 플랫폼이나 별도 메뉴를 만들지 않고, 현재 데이터와 대시보드 안에서 완성한다.

## 3. 우선순위와 단계

| 단계 | 기능 | 사용자 가치 | 배치 |
|---|---|---|---|
| 1차 | 시간대별 혼잡도·장비 가동률·취소율 | 조교가 운영 병목을 발견 | 관리자 대시보드 `운영 인사이트` |
| 1차 | 수요 증가·장비 부족·미반납 위험 | 문제가 커지기 전 처리 | 관리자 대시보드 `주의 필요` |
| 1차 | 그룹형 즐겨찾기·최근 예약·동일 구성 재예약 | 반복 예약 시간을 단축 | 학생 홈 `다시 예약` |
| 1차 | 빈도 제한 알림 | 대여·반납 누락을 줄임 | 학생 앱 알림 설정과 다음 예약 카드 |
| 2차 | 가능 날짜·시간·대체 장비 추천 | 예약 실패를 대안 선택으로 전환 | 예약 검증 오류 화면과 재예약 흐름 |

## 4. 1차 기능 상세

### 4.1 관리자 대시보드: 운영 인사이트

현재 대시보드의 처리 카드와 오늘 일정 아래에 `운영 인사이트` 블록을 둔다. 새 관리자 메뉴는 만들지 않는다.

- `혼잡 시간`: 최근 4주 동안 예약 가능 시간 대비 예약된 비율이 높은 상위 3개 시간대
- `장비 가동률`: 활성·예약 가능 장비별 예약된 일수 비율. 상위 5개 장비만 우선 표시
- `취소율`: 같은 기간 전체 신청 중 취소·관리자 취소·반려의 비율
- `주의 필요`: 수요 증가, 장비 부족, 미반납 위험을 최대 3개 카드로 요약

모든 카드에는 기존 관리 화면으로 연결되는 단일 행동을 둔다. 예를 들어 `Canon R5 가동률 92%`를 누르면 해당 장비 또는 예약 목록에 이미 적용된 필터로 이동한다.

### 4.2 학생 홈: 다시 예약

학생 홈의 `다음 예약` 아래에 작은 `다시 예약` 블록을 둔다. 블록의 `즐겨찾기 관리`를 누르면 전체 화면 전환 없이 바텀시트가 열린다.

- 최근 예약: 최근 완료 또는 확정된 예약 최대 3건
- 즐겨찾기 그룹: 학생이 직접 이름을 붙인 그룹 최대 3개. 그룹당 기자재 최대 5개
- 바텀시트: 그룹 선택·생성·이름 변경, 장비 검색, 추가·삭제를 한 곳에서 처리
- 홈 표시: 그룹별 대표 기자재 3개와 `더 보기` 수만 표시해 대시보드가 길어지지 않게 함
- 동일 구성 재예약: 날짜와 시간은 비우고, 기존 장비·시설·사용 목적 등 재사용 가능한 항목만 초안으로 복사

재예약은 기존 예약을 복제해 확정하지 않는다. 학생이 날짜·시간을 다시 고르고, 현재 재고·차단 일정·정책 검증을 통과해야 제출할 수 있다.

즐겨찾는 장비는 한 그룹에만 둘 수 있다. 장비를 다른 그룹에 추가하면 기존 그룹에서 이동시키며, 이미 그룹이 3개일 때는 새 그룹 생성 대신 기존 그룹 선택만 제공한다.

### 4.3 빈도 제한 예약 알림

기존 네이티브 알림 기능을 유지하되, 기본 알림을 줄인다.

- 예약 전날: 다음 날 예약을 한 번 안내
- 대여·사용 직전: 전날 알림과 12시간 이상 떨어질 때만 추가
- 반납 임박: 기자재 예약에만 우선 적용
- 12시간 이내에 겹치는 알림은 가장 행동이 급한 알림 하나로 합침
- 취소·반려·반납 완료 상태가 되면 해당 예약의 남은 알림을 제거

스튜디오 사용 보고서 알림은 현재 정책을 그대로 유지한다. 예약 알림과 보고서 마감 알림은 목적이 다르므로 서로 합치지 않는다.

## 5. 2차 기능 상세: 예약 불가 시 추천

추천은 학생 홈에 상시 노출하지 않는다. 예약하려는 장비·시간·공간이 실제로 불가능할 때와 `다시 예약` 과정에서만 보여 준다.

추천 순서는 다음과 같다.

1. 같은 장비를 쓸 수 있는 가장 가까운 날짜 또는 시간
2. 같은 카테고리에서 같은 조건으로 가능한 대체 장비
3. 같은 날짜에 가능한 다른 시간

추천 결과는 최대 3개만 보여 주고, 이유를 함께 쓴다. 예: `내일 13:00–15:00 가능`, `같은 카테고리 · Sony A7 IV 사용 가능`.

추천을 선택해도 서버의 예약 검증을 다시 실행한다. 추천 화면은 편의 기능일 뿐, 중복 예약·차단 일정·대여 정책을 우회하는 수단이 될 수 없다.

## 6. 제외 범위

이번 고도화에는 다음을 포함하지 않는다.

- 별도 통계 메뉴 또는 복잡한 BI 도구
- AI 모델 학습이나 개인별 행동 예측
- QR 인계·반납, 장비 상태 사진, 팀 예약
- 외부 캘린더 연동
- 학생에게 전체 혼잡도·다른 학생 예약 내역 공개
- 신규 예약 유형이나 신규 관리자 권한

## 7. 성공 기준

출시 후 4주 기준으로 다음을 확인한다.

| 관점 | 기준 |
|---|---|
| 학생 편의 | 최근 예약에서 다시 신청 시작까지 3회 이하의 행동으로 도달 |
| 즐겨찾기 | 홈의 바텀시트에서 최대 3개 그룹·그룹당 5개 장비 범위 안에서 관리 |
| 추천 품질 | 예약 불가 상황에서 최소 1개의 유효한 대안을 제시하거나, 대안이 없음을 명확히 표시 |
| 운영 대응 | 대시보드 경고에서 기존 예약·장비 관리 화면으로 한 번의 선택으로 이동 |
| 알림 품질 | 하나의 예약에 12시간 안으로 중복된 예약 알림이 없음 |
| 개인정보 | 집계 지표와 학생 추천 화면에 다른 학생의 이름·학번·연락처를 노출하지 않음 |

## 8. 완성 순서

1차에서 데이터 집계, 대시보드 인사이트, 다시 예약, 알림 제한을 먼저 검증한다. 실제 사용 데이터가 쌓인 뒤 2차에서 결정형 추천을 연결한다. 이 순서는 학생에게 불완전한 추천을 먼저 노출하지 않으면서, 운영자가 추천에 필요한 수요와 부족 상태를 먼저 확인하게 한다.

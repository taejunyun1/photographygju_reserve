# 스마트 예약 운영 대시보드 시뮬레이션

작성일: 2026-07-20

상태: 검토용 사용자 시나리오

연계 문서: [고도화 기능 기획서](2026-07-20-smart-reservation-feature-brief.md), [데이터·구현 로드맵](2026-07-20-data-and-delivery-roadmap.md)

## 1. 화면 배치 원칙

기능은 현재 정보 구조를 유지한다. 학생은 홈과 예약 화면만 사용하고, 조교는 기존 관리자 대시보드와 예약·장비 목록으로 이동한다.

```text
학생 홈
  다음 예약
  다시 예약 ─ 최근 예약 3건 / 즐겨찾는 기자재 5개
  공지사항 / 모집중 특강

예약 화면
  기존 단계형 예약 폼
  예약 불가일 때만: 가능한 날짜 · 시간 · 대체 장비

관리자 대시보드
  기존 처리 카드와 오늘 처리 큐
  운영 인사이트 ─ 혼잡 시간 / 가동률 / 취소율
  주의 필요 ─ 수요 증가 / 부족 위험 / 미반납 위험
```

## 2. 학생 시나리오

### 시나리오 A: 최근 기자재 예약을 다시 신청한다

| 단계 | 학생 화면 | 시스템 처리 |
|---|---|---|
| 1 | 홈의 `다시 예약`에서 지난 `Sony A7 IV · 렌즈 · 1박 2일` 카드를 선택 | 최근 확정 또는 완료된 예약만 최대 3건 표시 |
| 2 | 예약 초안이 열리고 장비·사용 목적·연락처가 채워짐 | 날짜·대여 시간·반납 시간은 비운 상태로 시작 |
| 3 | 학생이 새 날짜와 시간을 선택 | 현재 장비 상태, 차단 일정, 중복 예약, 대여 정책을 재검증 |
| 4 | `승인 요청` 선택 | 새 예약을 생성하고 기존 예약과 연결하지 않음 |

성공 상태: 학생은 장비를 다시 검색하지 않아도 되며, 이전 예약이 자동 확정되지는 않는다.

### 시나리오 B: 원하는 장비가 이미 예약되어 있다

| 단계 | 학생 화면 | 시스템 처리 |
|---|---|---|
| 1 | 학생이 날짜·시간·장비를 선택하고 제출 | 기존 예약 검증에서 중복을 감지 |
| 2 | 오류 문구 아래 `다른 선택지` 카드가 나타남 | 동일 장비의 가까운 가능 시간부터 탐색 |
| 3 | `내일 13:00–15:00 가능` 또는 `같은 카테고리 Sony A7 IV 가능`을 선택 | 해당 값만 예약 초안에 적용 |
| 4 | 학생이 검토 후 다시 제출 | 서버가 모든 정책을 다시 검증 |

대안이 없는 경우에는 `현재 조건에서 가능한 대안이 없습니다. 날짜나 시간 조건을 바꿔 주세요.`를 보여 준다. 존재하지 않는 추천을 만들지 않는다.

### 시나리오 C: 알림이 과도하지 않게 예약을 안내한다

학생이 내일 10:00에 기자재를 대여하고 17:00에 반납한다.

| 시점 | 기본 알림 | 조건 |
|---|---|---|
| 전날 | `내일 기자재 대여가 있습니다` | 항상 하나만 예약 |
| 대여 1시간 전 | `곧 기자재를 대여합니다` | 전날 알림과 12시간 이상 차이날 때만 |
| 반납 1시간 전 | `반납 예정 시간이 다가옵니다` | 대여 상태이며 아직 반납되지 않았을 때 |

학생이 예약을 취소하거나 조교가 반려하면 남은 예약 알림을 모두 취소한다. 반납 처리가 완료되면 반납 임박 알림도 취소한다.

### 시나리오 D: 즐겨찾는 장비에서 예약을 시작한다

학생은 자주 쓰는 카메라 2대를 즐겨찾기에 저장한다.

1. 학생 홈의 `즐겨찾는 기자재`에서 장비를 선택한다.
2. 예약 화면이 해당 장비가 미리 선택된 상태로 열린다.
3. 학생은 날짜와 대여 조건을 입력한다.
4. 장비가 수리중·비활성·예약 불가 상태라면 선택을 해제하고 이유를 표시한다.

즐겨찾기는 예약 가능을 보장하지 않는다. 저장해 둔 장비가 현재 대여 가능하지 않을 때는 상태를 정직하게 보여 준다.

## 3. 관리자 시나리오

### 시나리오 E: 조교가 혼잡 시간과 장비 부족을 확인한다

관리자 대시보드의 `운영 인사이트`에 다음 카드가 보인다.

| 카드 | 예시 표시 | 선택 시 이동 |
|---|---|---|
| 혼잡 시간 | `목요일 14:00–16:00 · 점유 88%` | 해당 날짜 범위·시간대의 예약 목록 |
| 장비 가동률 | `Canon R5 · 92%` | 해당 장비가 포함된 예약 또는 장비 상세 |
| 취소율 | `최근 4주 12%` | 취소·반려 예약 목록 |
| 부족 위험 | `조명 카테고리 3주 연속 고수요` | 조명 장비 목록과 관련 예약 |

조교는 카드에서 통계 화면으로 이동하지 않고 바로 처리해야 할 기존 화면에 도달한다. 통계는 판단의 출발점이며, 실무 처리는 기존 목록에서 계속한다.

### 시나리오 F: 반납 지연 위험을 처리한다

1. 기자재가 `checked_out` 상태이고 반납 예정 시간이 지났다.
2. 대시보드 `주의 필요`에 `반납 확인 필요 1건`이 나타난다.
3. 조교가 카드를 선택하면 해당 예약의 대여 중 필터가 열린다.
4. 반납을 확인하면 상태를 `returned`로 변경한다.
5. 대시보드 위험 카드와 학생의 남은 반납 알림이 다음 동기화에서 사라진다.

이 흐름은 자동 벌점이나 자동 취소를 수행하지 않는다. 실제 반납 여부는 조교가 확인한다.

## 4. 빈 상태와 오류 복구

| 상황 | 화면 문구와 동작 |
|---|---|
| 최근 예약 없음 | `최근 예약이 없습니다. 새 예약을 시작해 보세요.`와 예약 시작 버튼 |
| 즐겨찾기 없음 | `자주 쓰는 기자재를 저장하면 여기에서 바로 예약할 수 있습니다.` |
| 분석 데이터 부족 | `최근 4주 데이터가 충분하지 않아 추세를 표시하지 않습니다.` |
| 대안 없음 | 조건을 바꾸라는 안내만 표시하고 임의의 장비를 추천하지 않음 |
| 알림 권한 거부 | 앱 설정에서 알림을 켜는 방법을 안내하고 예약 기능은 계속 제공 |
| 통계 API 실패 | 기존 처리 카드와 예약 관리 기능은 계속 표시하고 인사이트 영역에 재시도 안내 |

## 5. 화면 검증 체크

- 390px와 430px 모바일 화면에서 `다시 예약` 카드와 다음 예약 카드가 겹치지 않는다.
- 관리자 모바일 화면에서는 인사이트 카드가 1열 또는 읽기 쉬운 2열로 배치되고, 카드 안의 수치가 잘리지 않는다.
- 인사이트 카드와 추천 카드의 단독 선택 요소는 44px 이상의 터치 영역을 가진다.
- 아이콘만 있는 즐겨찾기 토글에는 장비명과 현재 상태를 포함한 접근성 이름을 제공한다.
- 추천 결과는 최대 3개로 제한하고, 각 결과는 왜 추천되었는지 텍스트로 설명한다.

# Mobile Reservation Card Layout Design

Date: 2026-07-14
Status: Approved design

## Summary

학생 홈과 `내 예약`에서 공통으로 사용하는 예약 상태 카드를 모바일 우선 정보 구조로 다시 정렬한다. 상태 배지는 왼쪽, 취소 버튼은 오른쪽의 같은 첫 행에 고정한다. 제목은 현재보다 작은 17px로 낮추고, 날짜와 시간은 한 문장 안에 섞지 않고 별도 정보 행으로 구분한다. 긴 기자재 코드는 앞의 두 개만 표시하고 나머지는 `외 N개`로 요약한다.

`내 예약`의 여섯 카테고리 탭은 모바일에서 3열 2행으로 모두 노출한다. 탭 컨테이너의 내부 가로 스크롤 때문에 마지막 탭이 잘려 보이던 문제를 제거하고, 페이지와 카드의 실제 가로 폭도 viewport 안에 유지한다.

예약 데이터, 취소 API, 상태값, 검색 및 필터 동작은 변경하지 않는다.

## Evidence and Root Cause

390×844 환경에서 첨부된 TestFlight 상태를 재현했다.

- 문서 자체는 `390px`로 overflow가 없지만, 카테고리 탭의 표시 폭은 `311px`, 내부 콘텐츠 폭은 `426px`다. 처음 위치에서는 `출력실` 일부와 `특강` 전체가 컨테이너 밖에 있어 화면이 잘린 것처럼 보인다.
- 예약 카드의 header는 `display: block`이다. 배지 묶음이 `321×28px` 한 줄을 차지한 뒤 취소 버튼이 다음 줄 왼쪽에 `44×44px`로 배치된다.
- 카드 제목 `h2`는 브라우저 기본 상하 margin 약 `19.92px`를 유지한다. 범용 카드의 `10px` grid gap과 합쳐져 배지·취소 버튼·제목 사이에 불필요한 큰 공백을 만든다.
- `reservationMeta()`가 날짜, 시간, 기간, 모든 기자재 코드를 하나의 점 구분 문장으로 합쳐 빠른 확인과 긴 목록의 줄바꿈이 모두 불리하다.

## Goals

- 상태 배지와 취소 버튼을 첫 행의 양 끝에 한 줄로 정렬한다.
- 취소 버튼의 44px 터치 영역과 명확한 접근성 이름을 유지한다.
- 제목을 17px로 줄이고 기본 margin을 제거해 카드의 빈 공간을 줄인다.
- 날짜와 시간을 별도 값으로 명확하게 표시하고 기간은 보조 정보로 둔다.
- 기자재 코드는 최대 두 개만 보여주고 남은 수를 `외 N개`로 요약한다.
- 홈과 `내 예약`에서 같은 카드 표현을 사용한다.
- 390px과 430px에서 모든 필터를 잘림 없이 표시하고 body 가로 overflow를 만들지 않는다.

## Non-goals

- 예약 상태, 취소 가능 조건, API 변경
- 예약 생성 플로우나 하단 내비게이션 변경
- Admin 화면 변경
- 전체 학생 화면의 색상·서체 재설계
- 예약 상세 화면 추가

## Alternatives Considered

### A. Structured compact card — selected

첫 행에는 배지와 취소 버튼, 둘째 영역에는 17px 제목, 다음 영역에는 날짜·시간 정보, 마지막에는 기자재 요약을 둔다. 높이를 과도하게 늘리지 않으면서 사용자가 가장 자주 찾는 날짜와 시간을 빠르게 읽을 수 있다.

### B. Single-line metadata card

현재처럼 날짜·시간·기간·장비를 한 문장으로 두되 간격만 줄인다. 가장 낮은 높이를 만들 수 있지만 긴 장비 목록에서 위계와 가독성이 다시 무너진다.

### C. Status-accent card

카드 왼쪽 선이나 배경에 상태 색을 크게 적용한다. 상태 구분은 빠르지만 배지와 색상 의미가 중복되고 여러 카드가 쌓일 때 화면이 복잡해진다.

A안을 선택한다. 상태 색은 기존 배지에만 사용하고 카드 본체는 조용한 흰색 표면을 유지한다.

## Visual Direction

사진 촬영 일정표의 `날짜 / 시간 / 장비` 인덱스를 모티프로 삼는다. 장식보다 촬영에 필요한 핵심 정보의 읽는 순서를 디자인의 특징으로 사용한다.

```text
┌─────────────────────────────────┐
│ [기자재] [승인 대기]         [×] │
│ 기자재 예약                      │
│ ┌ 날짜 ──────┐ ┌ 시간 ──────┐ │
│ │ 7월 27일 월 │ │ 12:00–17:10│ │
│ └────────────┘ └────────────┘ │
│ 1박 2일 · CAM-750D-03,           │
│ LEN-1855-03 외 3개               │
└─────────────────────────────────┘
```

- 제목: `17px / 1.35 / 800`
- 날짜·시간 label: `11–12px / 700`, muted
- 날짜·시간 value: `14px / 750`, dark ink
- 기자재 요약: `13px / 1.45`, muted, 자연스러운 줄바꿈
- 기존 blue·amber·green·red 상태 배지와 흰색 카드 surface를 유지한다.

## Component Design

### ReservationCard

- `.student-react-reservation-card__head`를 2열 flex 또는 grid로 만들고 `align-items: center`, `justify-content: space-between`을 적용한다.
- 배지 묶음은 필요한 경우 내부에서만 줄바꿈하며, 취소 버튼은 `flex: 0 0 44px`로 오른쪽에 고정한다.
- 제목은 기본 margin을 제거하고 17px로 지정한다.
- `reservationMeta()`의 단일 문자열 대신 구조화된 표시 값을 만든다.
  - 날짜: `reservedDate` 또는 특강 날짜
  - 시간: `rentalTime–returnTime`, time slot, 또는 특강 시간
  - 기간: 값이 있을 때만 표시
  - 기자재: 코드 최대 두 개와 남은 개수
- 값이 없는 행은 렌더링하지 않는다. 모든 값이 없을 때만 기존의 최소 안내 문자열을 사용한다.
- 기존 보고서 작성 action과 취소 오류 영역은 카드 하단에 유지한다.

### Mine category filters

- 700px 이하에서 `.student-react-filter-tabs`를 3열 grid로 전환한다.
- 여섯 탭은 같은 폭과 최소 44px 터치 높이를 갖고 2행으로 모두 표시한다.
- 701px 이상은 기존 한 줄 horizontal tabs 패턴을 유지한다.
- tab role, roving keyboard 이동, `aria-selected`, 연결된 tabpanel 계약은 바꾸지 않는다.

## Responsive and Overflow Rules

- 카드, header, metadata의 각 자식에 `min-width: 0`과 `max-width: 100%`를 적용한다.
- 날짜·시간은 390px과 430px에서 2열을 기본으로 하되, 360px 이하에서는 1열로 접을 수 있다.
- 기자재 코드는 토큰 중간에서 억지로 잘리지 않도록 쉼표 뒤에서 우선 줄바꿈하고, 비정상적으로 긴 단일 코드는 `overflow-wrap: anywhere`로 보호한다.
- 탭 컨테이너는 모바일에서 `width: 100%`, `overflow: visible`을 사용하고 각 tab은 `min-width: 0`으로 둔다.
- 의도된 세로 스크롤 외에 document, body, 카드 내부에 가로 스크롤을 만들지 않는다.

## Accessibility and Interaction

- 취소 버튼은 현재의 `예약 취소` 또는 `특강 신청 취소` 접근성 이름과 44×44px 영역을 유지한다.
- 배지와 날짜·시간 값의 DOM 읽기 순서는 화면의 시각 순서와 일치한다.
- 날짜·시간의 작은 label은 색상만으로 구분하지 않고 텍스트로 명시한다.
- 탭의 keyboard interaction과 focus-visible 표시를 유지한다.
- 상태는 기존처럼 badge text로 전달하며 색상만으로 의미를 만들지 않는다.

## Testing and Verification

테스트를 먼저 추가하고 기대한 이유로 실패하는지 확인한 뒤 구현한다.

- 공통 예약 카드 header에서 배지 묶음과 취소 버튼이 같은 y축 행에 있는지 검사한다.
- 취소 버튼의 오른쪽 경계가 카드 content 오른쪽에 정렬되고 크기가 44×44px인지 검사한다.
- 제목 계산 font-size가 17px이고 불필요한 기본 margin이 없는지 검사한다.
- 다섯 기자재 입력에서 두 코드와 `외 3개`만 표시되고 나머지 전체 코드가 노출되지 않는지 검사한다.
- 날짜와 시간이 각각 식별 가능한 label/value로 표시되는지 검사한다.
- 390px과 430px에서 여섯 탭이 모두 컨테이너 안에 있고 탭 container의 `scrollWidth <= clientWidth + 1`인지 검사한다.
- 홈과 `내 예약` 양쪽에 동일한 카드 계약을 검사한다.
- document/body horizontal overflow가 없는지 검사한다.
- serious/critical axe 위반, 기존 학생 계약, bridge, UI 회귀 테스트와 build를 실행한다.

## Acceptance Criteria

1. 상태 배지와 취소 버튼이 첫 행의 좌우에 정렬된다.
2. 취소 버튼은 오른쪽에 고정되며 44×44px 터치 영역을 유지한다.
3. 카드 제목은 17px이고 배지와 제목 사이의 불필요한 빈 공간이 사라진다.
4. 날짜와 시간은 서로 분리된 label/value로 표시된다.
5. 기자재가 세 개 이상이면 앞의 두 코드와 `외 N개`로 요약된다.
6. 홈과 `내 예약` 카드가 같은 표현을 사용한다.
7. 모바일에서 여섯 필터가 3열 2행으로 모두 보이고 내부 가로 스크롤이 없다.
8. 390px과 430px에서 화면 바깥 overflow가 없다.
9. 기존 취소, 보고서, 필터, keyboard, screen reader 동작이 유지된다.

## Implementation Boundaries

변경은 `StudentPrimitives`, 학생 전용 CSS, 필요한 학생 UI 회귀 테스트에 한정한다. 새 라이브러리나 서버 상태를 추가하지 않는다. 범용 디자인 시스템 탭은 다른 화면의 동작을 바꾸지 않도록 학생 전용 class에서만 재배치한다.

# 관리자 대시보드 정량 지표 정합성 설계

## 목표

관리자 대시보드의 카드와 운영 지표가 실제로 처리 가능한 예약 및 연결된 목록과 같은 기준을 사용하도록 한다. 별도 통계 화면이나 기간 선택 기능은 추가하지 않고 기존 대시보드 안에서 최소 변경한다.

## 집계 기준

### 대여/반납 큐

- 오늘 대여 시작 예정인 `approved` 기자재 예약은 `checkout` 항목으로 표시한다.
- 이미 대여된 `checked_out` 예약은 오늘 반납 예정일 때만 `return` 항목으로 표시한다.
- 같은 예약을 `checkout`과 `return`으로 동시에 표시하지 않는다.
- 모든 날짜 판정은 `Asia/Seoul` 기준을 사용한다.

### 기자재 가용률

- 전체는 비활성화되지 않은 기자재로 계산한다.
- 가능 수량은 `status === "가능"`이면서 `reservable !== false`이고 `inquiryOnly !== true`인 기자재로 계산한다.
- 따라서 문의 전용과 판타지랩 직접 문의 장비는 가능 수량에서 제외한다.

### 예약 통계

- 이번 주 예약 수, 예약 유형별 비중, 인기 기자재 Top 5에서 `cancelled`, `admin_cancelled`, `rejected` 상태를 제외한다.
- 이번 주 예약 수에는 승인 대기·승인·대여 중뿐 아니라 실제 사용이 끝난 `returned`, `completed`도 포함한다.
- 인기 기자재는 유효 예약에서 선택된 장비 이름별 선택 횟수를 계산한다.
- 취소율은 기존처럼 최근 28일 전체 요청 중 취소·관리자 취소·반려 비율을 유지한다.
- 장비 가동률은 기존처럼 최근 28일 중 예약된 고유 날짜 비율을 유지한다.

### 보고서 확인 큐

- 새 스튜디오 보고서는 `submitted` 상태로 저장한다.
- 관리자 보고서 화면에서 `submitted` 보고서를 `reviewed`로 확인 완료할 수 있다.
- 보고서 확인 큐는 상태가 없거나 `submitted`인 보고서만 계산한다.
- `reviewed`, `rejected`, `approved`, `completed` 보고서는 큐에서 제외한다.
- 기존 상태 없는 보고서는 호환성을 위해 `submitted`로 간주한다.

### 오늘 처리 카드

- `반납 완료`와 `취소/반려` 카드는 서울 기준 오늘 예약 건수만 표시한다.
- 두 카드를 누르면 같은 오늘 날짜 범위와 상태 필터가 적용된 예약 목록으로 이동한다.
- 승인 대기, 승인 완료, 대여 중 카드는 현재 처리 상태 전체를 보여주는 기존 기준을 유지한다.

## API와 화면 변경

- `GET /api/admin/summary`가 위 기준으로 카드, 큐, 운영 지표를 계산한다.
- `PATCH /api/admin/reports/:id/status`는 관리자만 호출할 수 있고 `reviewed` 상태 변경만 허용한다.
- 보고서 화면의 제출 상태 행에는 `확인 완료` 버튼을 제공하며 성공 후 보고서 목록과 대시보드 요약을 새로 불러온다.
- 기존 보고서 삭제, 검색, 학기 및 상태 필터 동작은 유지한다.

## 오류 처리

- 존재하지 않는 보고서는 `404`를 반환한다.
- 허용하지 않은 상태는 `400`을 반환한다.
- 관리자 이외 호출은 기존 인증 규칙대로 거부한다.
- 확인 처리에 실패하면 기존 관리자 알림 방식으로 오류를 표시하고 목록 상태는 변경하지 않는다.

## 테스트

- 동일 날짜의 `approved`와 `checked_out` 예약을 사용해 대여/반납 큐 중복 및 누락 방지 테스트를 추가한다.
- 문의 전용 장비가 가용 수량에서 제외되는지 테스트한다.
- 취소·반려 예약이 주간·유형·인기 통계에서 제외되는지 테스트한다.
- 보고서 생성 기본 상태, 관리자 확인 API, 확인 큐 감소를 테스트한다.
- 오늘 반납·취소 카드와 클릭 후 목록 필터의 건수가 일치하는지 테스트한다.
- 기존 대시보드 정합성, 백엔드 도메인, 관리자 UI 및 전체 릴리스 검증을 실행한다.

## 범위 제외

- 별도 통계 페이지
- 사용자 지정 기간 선택
- 보고서 반려 및 재제출 워크플로
- 시간 단위 장비 가동률
- 과거 데이터 일괄 마이그레이션

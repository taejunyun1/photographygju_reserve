# 학년·학기별 교과 수요조사 빌더 설계

작성일: 2026-07-20  
상태: 사용자 승인 설계  
대상 앱: GJU Photography Reservation 관리자·학생 대시보드

## 1. 목적

학과 관리자가 학교 학사 시스템의 편성안을 대신 만들지 않고, 매 학기 말 다음 학기 전공선택 과목의 수요를 조사할 수 있게 한다. 관리자는 학년·학기를 지정하고 전선 과목 5~6개를 골라 설문안을 공개한다. 학생은 자신의 대상 설문에서 희망 과목을 최대 5개까지 선택해 1~5순위를 제출한다.

이 기능의 핵심은 다음 세 가지다.

1. 편성안 없이 독립적으로 설문안을 생성·공개·마감한다.
2. 과목을 `예술`, `다큐멘터리`, `광고`, `영상` 카테고리로 구분해 학생이 후보의 성격을 이해하고 투표한다.
3. 관리자는 개인 응답이 아닌 익명 집계로 과목별·카테고리별 수요를 확인한다.

## 2. 범위

### 포함

- 관리자 설문안 생성·임시저장·공개·마감
- 학년도, 학기, 응답 대상 현재 학년, 다음 수강 학년, 응답 기간 설정
- 조건에 맞는 전선 과목 필터링과 후보 5~6개 선택
- 과목별 수요 카테고리 관리
- 학생의 1~5순위 응답과 마감 전 수정
- 응답률, 과목별 선택 인원·가중 점수·순위별 득표수, 카테고리별 총수요 집계
- 기존 설문·응답 데이터와의 호환

### 제외

- 학교 학사 시스템의 학년도 편성안 작성
- 연간 85학점·교육과정 130학점 계산 및 편성안 확정
- 전필·2년 주기 과목의 개설 검증
- 교수·강의실·시간표 배정
- 자동 개설 추천
- 수강신청과 수강 확정

기존 편성안 도메인 데이터와 API는 저장 데이터 호환을 위해 즉시 삭제하지 않지만, 관리자 사용자 화면과 새 설문 생성 흐름에서는 사용하지 않는다.

## 3. 사용자 흐름

### 3.1 관리자

관리자 메뉴명은 `교과 편성`에서 `교과 수요조사`로 변경한다. 화면은 다음 세 탭으로 단순화한다.

1. `설문안`: 새 설문 작성, 임시저장, 공개, 마감, 기존 설문 상태 확인
2. `과목 관리`: 과목명·코드·대상 학년·학기·전필/전선·수요 카테고리·활성 상태 관리
3. `결과`: 설문별 응답률과 익명 수요 집계 확인

새 설문안은 다음 순서로 만든다.

1. 설문 제목, 학년도, 학기, 응답 대상 현재 학년, 다음 수강 학년, 시작·마감 시각을 지정한다.
2. 시스템은 활성 상태이며 설문 대상이고, 전선이며, 다음 수강 학년과 학기에 맞는 과목만 후보로 보여준다.
3. 관리자는 카테고리 필터를 활용해 후보 과목을 선택한다. 카테고리별 최소 선택 수는 없다.
4. 임시저장 상태에서는 후보 0~6개를 저장할 수 있다.
5. 공개할 때는 후보가 정확히 5개 또는 6개여야 한다.
6. 공개된 설문은 학생 응답의 의미를 보호하기 위해 과목 후보와 대상 학년·학기를 수정할 수 없다. 마감 시각 연장과 설문 마감만 허용한다.

동일한 학년도·학기·응답 대상 학년에 공개 중인 설문은 하나만 허용한다. 중복 공개 시 서버가 거부하고 관리자 화면에 원인을 표시한다.

### 3.2 학생

1. 승인된 학생의 홈 대시보드에 본인 학년에 해당하는 공개 설문을 표시한다.
2. 설문 화면은 후보 5~6개를 `예술`, `다큐멘터리`, `광고`, `영상` 그룹으로 나누어 보여준다. 후보가 없는 카테고리는 표시하지 않는다.
3. 카테고리는 이해를 돕는 분류이며 카테고리별 의무 선택은 없다.
4. 학생은 최소 1개, 최대 5개 과목을 선택하고 1순위부터 빈칸 없이 정렬한다.
5. 마감 전에는 마지막 응답을 수정할 수 있다. 마감 후에는 제출 여부만 확인한다.
6. 화면에는 이 설문이 수강신청이나 개설 보장이 아니라는 안내를 표시한다.

## 4. 카테고리 정책

과목의 수요 카테고리는 다음 네 값 중 하나다.

| 저장 값 | 표시명 | 예시 성격 |
| --- | --- | --- |
| `art` | 예술 | 예술사진, 창작, 이미지 연구 |
| `documentary` | 다큐멘터리 | 다큐멘터리, 포토스토리, 포토에세이 |
| `advertising` | 광고 | 커머셜, 라이팅, 스튜디오, 광고 제작 |
| `video` | 영상 | 영상 촬영·편집, 드론, 영상 콘텐츠 |

카테고리는 과목 마스터의 `demandCategory` 필드로 저장한다. 설문 공개 시 과목명·학점·대상 정보와 함께 스냅샷에 복사한다. 이후 과목 마스터의 카테고리가 변경되어도 이미 공개된 설문의 분류는 바뀌지 않는다.

초기 시드 과목에는 이름과 교육과정 성격에 따라 네 카테고리를 명시적으로 부여하고, 관리자가 `과목 관리` 탭에서 수정할 수 있게 한다. 카테고리가 없는 과목은 설문 후보로 선택할 수 없으며 `카테고리 지정 필요`로 표시한다.

## 5. 설문 상태와 검증

| 상태 | 허용 동작 | 후보 수 규칙 |
| --- | --- | --- |
| `draft` | 전체 내용 편집, 임시저장, 공개 | 0~6개 저장, 공개 시 5~6개 |
| `open` | 마감 시각 연장, 마감 | 5~6개 고정 |
| `closed` | 결과 열람 | 후보·대상·기간 수정 불가 |

서버는 다음을 검증한다.

- 시작 시각은 마감 시각보다 빨라야 한다.
- 공개 설문의 후보는 중복 없이 5~6개여야 한다.
- 모든 후보는 활성 전선 과목이며 대상 학년·학기와 일치해야 한다.
- 모든 후보에는 유효한 수요 카테고리가 있어야 한다.
- 응답 과목은 설문 스냅샷에 있어야 한다.
- 학생 응답은 1~5개이며 순위는 1부터 연속해야 한다.
- 대상 학년이 아니거나 설문 기간 밖이면 응답을 거부한다.
- 한 학생은 설문당 마지막 응답 하나만 갖는다.

## 6. 데이터 모델

### 과목

기존 과목 레코드에 다음 필드를 사용하거나 추가한다.

- `majorType`: `전필` 또는 `전선`
- `targetYears`: 대상 학년 배열
- `allowedTerms`: `spring`, `fall`
- `demandCategory`: `art`, `documentary`, `advertising`, `video`
- `isSurveyEligible`: 설문 후보 사용 여부
- `active`: 활성 여부

### 설문

설문은 편성안과 독립된 다음 필드를 가진다.

- `id`
- `title`
- `academicYear`
- `term`
- `eligibleCurrentYears`
- `targetStudentYears`
- `opensAt`, `closesAt`
- `status`
- `catalogSnapshot`
- `createdAt`, `updatedAt`

`semesterPlanId`는 이전 데이터 호환을 위한 선택 필드로만 유지하며 새 설문 생성에 요구하지 않는다.

### 설문 후보 스냅샷

각 후보는 다음 정보를 저장한다.

- `id`, `courseCode`, `name`
- `studentCredit`
- `targetYears`, `allowedTerms`
- `demandCategory`

## 7. API

### 관리자

- `GET /api/admin/course-planning`: 과목, 설문, 익명 요약을 조회한다. 기존 편성안 데이터는 응답 호환을 위해 유지하되 새 화면에서는 사용하지 않는다.
- `PUT /api/admin/courses`: 과목의 카테고리와 설문 필터 필드를 저장한다.
- `POST /api/admin/course-demand-surveys`: 독립 설문안을 생성한다. `courseIds`로 후보를 지정하며 `semesterPlanId`는 필요하지 않다.
- `PUT /api/admin/course-demand-surveys/:id`: 임시저장 설문 전체 편집, 공개 설문의 마감 시각 연장, 설문 마감을 처리한다.
- `GET /api/admin/course-demand-surveys/:id/summary`: 개인 식별 정보가 없는 결과를 반환한다.

설문 생성·공개·마감과 과목 카테고리 변경은 감사 로그에 기록한다.

### 학생

- `GET /api/me/course-demand-surveys`: 현재 학생에게 해당하는 설문과 본인 제출 상태를 조회한다.
- `PUT /api/me/course-demand-surveys/:id/response`: 본인의 1~5순위 응답을 저장하거나 수정한다.

## 8. 집계

순위별 가중치는 기존 정책을 유지한다.

| 순위 | 점수 |
| --- | ---: |
| 1순위 | 5 |
| 2순위 | 4 |
| 3순위 | 3 |
| 4순위 | 2 |
| 5순위 | 1 |

관리자 결과 화면은 다음을 표시한다.

- 대상 학생 수, 응답 학생 수, 응답률
- 과목별 선택 학생 수
- 과목별 가중 수요 점수
- 과목별 1~5순위 득표수
- 카테고리별 선택 수와 가중 수요 점수

학생 이름, 학번, 개별 순위 목록은 관리자 API와 화면에 제공하지 않는다.

## 9. UI 구성

### 설문안 탭

- 설문 기본 정보 입력 영역
- 학년·학기 조건에 따라 필터링된 전선 과목 목록
- 카테고리 필터와 과목별 카테고리 표시
- 선택된 후보 수 `n/6` 표시
- 임시저장과 공개 버튼
- 설문 상태 목록과 마감 동작

### 과목 관리 탭

- 과목명·코드
- 전필/전선
- 대상 학년과 허용 학기
- 수요 카테고리
- 설문 후보 사용 여부와 활성 상태

### 결과 탭

- 설문 선택
- 응답률 요약
- 카테고리별 수요 요약
- 과목별 점수·선택 수·순위 분포

### 학생 바텀시트

- 설문 제목과 대상 학기
- 카테고리별 후보 목록
- 선택한 과목 1~5순위 목록과 위·아래 이동
- 저장 상태와 오류 안내

기존 Astryx 기반 카드·버튼·탭·바텀시트 패턴을 재사용한다. 새로운 시각 체계나 장식 요소는 추가하지 않는다.

## 10. 오류 처리

- 후보가 5개 미만 또는 6개 초과이면 공개 버튼을 비활성화하고 정확한 수량 안내를 보여준다.
- 대상 조건에 맞는 전선 과목이 5개 미만이면 과목 관리에서 대상 학년·학기·카테고리를 먼저 보완하도록 안내한다.
- 서버 저장 실패 시 입력과 선택 상태를 유지하고 오류를 알린다.
- 공개 충돌, 마감 후 수정, 대상 불일치 등 서버 검증 메시지를 사용자용 문구로 표시한다.
- 공개 중 과목 마스터를 수정해도 설문 스냅샷과 기존 응답은 유지한다.

## 11. 테스트와 완료 기준

### 도메인·API

1. 편성안 없이 학년·학기와 `courseIds`만으로 설문안을 생성할 수 있다.
2. 임시저장은 후보 0~6개를 허용하고 공개는 정확히 5~6개만 허용한다.
3. 전필, 비활성, 대상 학년·학기 불일치, 카테고리 누락 과목의 공개를 거부한다.
4. 공개 설문의 후보 스냅샷과 대상 조건은 변경할 수 없다.
5. 학생은 대상 설문에만 1~5순위를 제출·수정할 수 있다.
6. 과목별·카테고리별 집계가 동일한 응답에서 일관되게 계산된다.

### 관리자 UI

1. 편성안 탭과 85·130학점 UI가 보이지 않는다.
2. 학년·학기 선택에 맞는 전선 과목과 카테고리만 표시된다.
3. 선택 수 `n/6`, 임시저장, 5~6개 공개 조건이 동작한다.
4. 설문 공개·마감·결과 확인을 같은 메뉴에서 완료할 수 있다.

### 학생 UI

1. 대시보드에서 본인 대상의 공개 설문으로 진입한다.
2. 후보가 카테고리별로 구분된다.
3. 카테고리 의무 선택 없이 최대 5개를 1~5순위로 저장한다.
4. 모바일에서 후보 선택, 순위 이동, 저장, 오류 복구가 가능하다.

### 완료 기준

- 관리자는 편성안을 만들지 않고 학년·학기별 전선 5~6개로 설문안을 공개할 수 있다.
- 학생은 카테고리를 확인하며 희망 과목을 1~5순위로 제출할 수 있다.
- 관리자는 개인 식별 정보 없이 과목·카테고리별 수요와 응답률을 확인할 수 있다.
- 기존 예약 기능과 기존 설문 응답 데이터가 깨지지 않는다.

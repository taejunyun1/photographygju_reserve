# Admin 설정 달력 및 학기 종료 정리 설계

## 목표

Admin 설정 화면에서 차단 일정을 달력 중심으로 빠르게 등록하고, 학기 종료 시 예약 정보와 모든 로그인 세션을 명확하고 안전하게 정리한다.

## 설정 화면 순서

1. 학기 차단 캘린더
2. 수업/학기 차단 일정 입력 및 등록 목록
3. 운영 설정
4. 알림 설정
5. Slack 안내
6. 보안 / 데이터 관리

달력의 이전 달과 다음 달은 아이콘형 작은 버튼으로, 오늘은 작은 텍스트 버튼으로 배치한다. 날짜를 선택하면 차단 일정 폼의 시작일과 종료일에 선택 날짜를 입력하고, 요일을 자동 선택한 뒤 폼으로 스크롤하고 첫 입력 요소에 포커스를 둔다. 날짜 선택만으로 저장하지 않으며 관리자가 시간과 시설을 확인한 뒤 등록한다.

## 학기 종료 정리

기존 `보관정책 정리`는 오래된 개인정보 익명화와 만료 데이터 정리에 계속 사용한다. 별도의 `학기 종료 데이터 정리`를 제공하며, 실행 전에 관리자가 `학기 종료`를 정확히 입력해야 한다.

서버는 하나의 관리 API 요청에서 다음 작업을 처리한다.

- 전체 예약 삭제
- 삭제된 예약에 연결된 보고서 삭제
- 관리자 본인을 포함한 전체 로그인 세션 해제 및 삭제
- 삭제 건수와 실행 관리자 ID를 활동 로그에 기록

작업 성공 후 현재 브라우저의 인증 정보도 제거하고 로그인 화면으로 이동한다. 버튼을 누르기 전에는 기존 데이터에 변화가 없다.

## 오류 처리

- Admin 권한이 아니면 요청을 거부한다.
- 확인 문구가 `학기 종료`와 다르면 요청을 거부한다.
- 서버 작업이 실패하면 로컬 인증 정보를 유지하고 오류 토스트를 표시한다.
- 성공 응답을 받은 경우에만 로컬 세션을 제거한다.

## 검증

- 설정 화면 렌더링 계약 테스트로 섹션 순서, 작은 달력 이동 버튼, 클릭 가능한 날짜, 명확한 보안 문구를 검증한다.
- 보안 API 테스트로 잘못된 확인 문구 거부, 예약·연결 보고서·전체 세션 삭제, 활동 로그 생성을 검증한다.
- 모바일 설정 화면에서 달력 버튼 크기, 날짜 선택 후 폼 이동과 값 자동 입력을 확인한다.

# 교과목 수요조사 및 학년도 편성 지원 기능 설계

작성일: 2026-07-20
상태: 사용자 검토용 설계 확정본
대상 앱: GJU Photography Reservation 관리자·학생 대시보드

## 1. 목적과 범위

매 학기 말 학생이 다음 학기에 수강하고 싶은 과목을 1~5순위로 응답하게 하고, 학과 관리자가 그 결과를 근거로 학년도 개설안을 작성하도록 지원한다. 이 기능은 수강신청 시스템이 아니라 **개설 편성 의사결정 도구**다. 학생 응답은 개설을 보장하지 않으며, 최종 편성 권한은 관리자에게 있다.

이번 범위는 다음 세 부분으로 나눈다.

1. 학생 수요조사: 다음 학기 수강 대상 학년에게만 과목 후보를 보여주고 1~5순위를 제출받는다.
2. 관리자 편성안: 과목별 수요·필수 규칙·학점 제약을 보면서 1·2학기 통합 개설안을 작성한다.
3. 규칙 검증: 전필, 2년 주기 의무 과목, 130학점 교육과정 한도, 연간 85학점 운영 한도, 현장실습·캡스톤 예외를 분리 계산한다.

수강신청, 시간표 자동배정, 강의실 배정, 교수 배정 확정, 학생별 졸업요건 판정은 이번 범위에서 제외한다.

## 2. 확정된 정책과 우선순위

### 2.1 정책의 출처

| 우선순위 | 출처 | 적용 방식 |
| --- | --- | --- |
| 1 | 학과 운영자가 이번 대화에서 확정한 규칙 | 시스템의 강제 검증 규칙 |
| 2 | 첨부 PDF `광주대 커리큘럼 정리_20251001.numbers-2026학년도 개설 예정` | 초기 과목 마스터 시드와 학년·학기 후보 |
| 3 | 관리자가 편성안에서 지정한 학기·학점·개설 상태 | 해당 학년도 편성안의 최종 값 |

첨부 PDF에는 `사진교과교육론`, `사진교수학습방법`이 매학기처럼 보이지만, 이 설계에서는 운영자가 확정한 규칙을 우선 적용한다. 두 과목은 각각 최근 2개 학년도 안에 최소 1회만 개설하면 된다.

### 2.2 용어

| 용어 | 정의 |
| --- | --- |
| 교육과정 편성학점 | 학과 교육과정 버전이 편성할 수 있는 총 학점 한도. 기본값 130학점. |
| 연간 운영학점 | 해당 학년도 1·2학기에 실제로 일반 운영 과목으로 개설할 수 있는 합계 한도. 하드 한도 85학점. |
| 학생인정학점 | 학생의 수강·성적·졸업 관련 학점. |
| 교수인정시수 | 교수 시수 집계에 쓰는 인정 학점. |
| 운영학점 | 연간 85학점 계산에 반영되는 학점. 학생인정학점·교수인정시수와 독립적으로 저장한다. |
| 학년도 편성안 | 특정 학년도의 1·2학기 과목 개설 초안과 최종안을 함께 보관하는 단위. |
| 수요조사 | 특정 편성안의 한 학기를 대상으로 하는 학생 순위 응답 기간. |

## 3. 교과 운영 규칙

### 3.1 학점·개설 제약

| 규칙 | 시스템 처리 |
| --- | --- |
| 교육과정 편성 총량은 130학점 | 교육과정 버전에서 `curriculumCreditLimit = 130`으로 저장한다. 교육과정에 포함되는 과목의 학생인정학점 합계를 별도 검증한다. 연간 개설안의 85학점과 합산하지 않는다. |
| 1·2학기 통합 운영 한도는 85학점 | 편성안에서 `operatingCredit` 합계가 85를 넘으면 확정할 수 없다. 학기별 목표치는 관리자가 설정할 수 있으나, 학년도 합계 85가 하드 한도다. |
| 전필은 반드시 개설 | `major_required = true` 과목은 해당 대상 학기 편성안에 자동 필수 후보로 넣고 제외를 막는다. 정말 개설 불가한 경우 관리자 사유·승인 기록이 있어야 한다. |
| 사진교과교육론·사진교수학습방법 | 각 과목은 최근 2개 학년도 안의 확정 또는 실제 개설 이력을 검사한다. 미개설이면 다음 적합 학기의 필수 후보가 된다. 초안·취소 과목은 이력으로 인정하지 않는다. |
| 캡스톤디자인 | 연간 85학점에는 포함하지 않는다. 교수인정시수에는 포함한다. 초기 시드는 학생 3학점, 교수 3시수, 운영 0학점으로 둔다. |
| 현장실습1·2 등 | 방학 운영 과목이다. 연간 85학점에는 포함하지 않는다. 교수인정시수는 과목 마스터에서 관리자가 확인·입력한다. |
| 현장실습4 | 4학년 2학기 한 학기 운영이다. 학생인정학점 15, 교수인정시수 3, 운영학점 0으로 고정한다. |

### 3.2 과목별 세 개의 학점 필드

모든 과목은 아래 세 값을 별도로 가진다. 단일 `credits` 필드 하나로는 예외 규칙을 안전하게 계산할 수 없다.

| 필드 | 일반 3학점 과목 | 캡스톤디자인 | 현장실습4 |
| --- | ---: | ---: | ---: |
| `student_credit` | 3 | 3 | 15 |
| `operating_credit` | 3 | 0 | 0 |
| `faculty_recognized_credit` | 3 | 3 | 3 |

현장실습1·2 등 방학 과목은 `operating_credit = 0`이다. 학생·교수 학점은 과목 마스터의 승인값을 사용한다.

## 4. 사용자 경험

### 4.1 관리자 흐름

1. 관리자가 학년도 편성안을 만든다. 이때 1·2학기 운영 한도 85학점과 선택적 학기별 목표치를 설정한다.
2. 교육과정 버전과 과목 마스터에서 해당 학기·대상 학년에 맞는 후보 과목을 불러온다.
3. 전필과 2년 주기 도래 과목은 자동으로 `필수 후보`로 표시한다.
4. 수요조사 기간, 대상 학년, 안내문을 설정해 공개한다.
5. 마감 뒤 대시보드에서 응답률, 순위별 수요, 학년별 수요, 과목별 점수, 85학점 잔여량을 확인한다.
6. 시스템이 만든 추천 순서를 참고해 1·2학기 개설 초안을 편집한다. 관리자의 추가·제외·학기 변경은 사유와 함께 기록한다.
7. 모든 하드 규칙을 통과하면 편성안을 확정한다. 확정된 과목만 다음 2년 주기 이력에 반영한다.

### 4.2 학생 흐름

1. 학생 대시보드에 설문 기간 배너와 `다음 학기 희망 과목 조사` 진입 버튼을 표시한다.
2. 학생은 승급 후 다음 학기에 수강 가능한 학년·학기 과목만 본다. 예를 들어 2학기 말 현재 2학년 학생에게는 다음 학년 3학년 1학기 후보를 보여준다.
3. 과목을 최대 5개까지 선택하고 1순위부터 5순위까지 정렬한다. 최소 1개, 최대 5개이며 중복·순위 공백은 허용하지 않는다.
4. 설문 마감 전에는 언제든 수정·재제출할 수 있으며 마지막 제출본만 집계한다.
5. 마감 뒤에는 본인 응답만 읽을 수 있다. 과목별 개설 확정 여부나 다른 학생의 세부 응답은 보이지 않는다.

## 5. 수요 점수와 편성 초안

### 5.1 집계 방식

| 학생 순위 | 점수 |
| --- | ---: |
| 1순위 | 5점 |
| 2순위 | 4점 |
| 3순위 | 3점 |
| 4순위 | 2점 |
| 5순위 | 1점 |

과목별로 다음 값을 함께 보여준다.

- 총 응답 학생 수와 응답률
- 선택 학생 수
- 순위별 선택 건수
- 가중 수요 점수
- 대상 학년별 선택 학생 수와 점수
- 과거 같은 과목의 확정 개설 이력

수요 점수는 동점일 때 우선순위를 정하는 보조 지표다. 전필·2년 주기 도래 과목을 대체하거나 자동으로 폐강시키지 않는다.

### 5.2 추천 순서

편성 초안 생성기는 다음 순서로 과목을 처리한다.

1. 해당 학기 전필을 먼저 포함한다.
2. 최근 2개 학년도에 개설 이력이 없는 사진교과교육론·사진교수학습방법을 적합한 학기에 포함한다.
3. 현장실습4, 방학 현장실습, 캡스톤디자인처럼 운영학점 0인 과목은 85학점 잔여량과 별도로 반영한다.
4. 나머지 후보를 가중 수요 점수 내림차순으로 제안한다.
5. 새 과목의 운영학점을 더했을 때 85학점을 넘으면 `한도 초과 후보`로 보류한다.

추천 결과는 초안일 뿐이다. 관리자는 과목을 이동·추가·제외할 수 있고, 시스템은 수정 뒤 즉시 학점·필수 규칙을 다시 계산한다.

### 5.3 검증 결과

| 상태 | 의미 | 확정 가능 여부 |
| --- | --- | --- |
| 통과 | 모든 하드 규칙 충족 | 가능 |
| 경고 | 응답률 부족, 수요 낮음, 선택적 학기 목표 초과 등 | 가능, 사유 권장 |
| 오류 | 85학점 초과, 전필 누락, 2년 주기 과목 누락, 학기·학년 불일치 | 불가 |

## 6. 데이터 모델

### 6.1 과목·교육과정

| 엔터티 | 핵심 필드 | 책임 |
| --- | --- | --- |
| `curriculum_versions` | `academic_year`, `curriculum_credit_limit`, `status` | 130학점 교육과정 한도와 버전 관리 |
| `courses` | `course_code`, `name`, `major_type`, `category`, `target_years`, `allowed_terms` | 과목 마스터 |
| `course_credit_rules` | `student_credit`, `operating_credit`, `faculty_recognized_credit`, `counts_toward_curriculum_130` | 세 종류 학점 계산 |
| `course_offering_rules` | `is_major_required`, `required_frequency_years`, `delivery_period`, `is_survey_eligible` | 전필·2년 주기·방학·설문 노출 규칙 |

`course_code`는 PDF에 없으므로 초기 등록 전에 관리자가 부여하거나 기존 학사 코드와 매핑해야 한다. 과목명은 코드의 보조 표시값일 뿐, 규칙 판정의 기본키가 아니다.

### 6.2 설문·편성

| 엔터티 | 핵심 필드 | 책임 |
| --- | --- | --- |
| `annual_offering_plans` | `academic_year`, `operating_credit_limit=85`, `status` | 1·2학기 통합 85학점 편성 단위 |
| `semester_offering_plans` | `annual_plan_id`, `term`, `optional_credit_target` | 학기별 후보·확정 과목 |
| `course_demand_surveys` | `semester_plan_id`, `eligible_current_years`, `opens_at`, `closes_at`, `status`, `catalog_snapshot` | 수요조사 기간과 과목 스냅샷 |
| `course_demand_responses` | `survey_id`, `student_id`, `submitted_at` | 학생당 하나의 마지막 응답 |
| `course_demand_rankings` | `response_id`, `course_id`, `rank` | 1~5순위 선택 |
| `planned_course_offerings` | `semester_plan_id`, `course_id`, `status`, `override_reason`, `source` | 추천·관리자 수정·확정 결과 |
| `course_offering_history` | `course_id`, `academic_year`, `term`, `status` | 확정·실제 개설 이력과 2년 규칙 판정 |

설문은 `catalog_snapshot`을 보관한다. 설문 진행 중 과목 마스터가 바뀌어도 이미 공개된 후보와 학생 응답의 의미가 달라지지 않게 하기 위함이다.

## 7. API와 권한 경계

### 학생 API

| 메서드·경로 | 기능 |
| --- | --- |
| `GET /api/me/course-demand-surveys` | 현재 학생에게 열린 설문과 본인 응답 조회 |
| `PUT /api/me/course-demand-surveys/:surveyId/response` | 1~5순위 응답 저장 또는 수정 |

학생 API는 로그인한 본인의 응답만 반환한다. 다른 학생의 이름, 학번, 개별 순위는 절대 반환하지 않는다.

### 관리자 API

| 메서드·경로 | 기능 |
| --- | --- |
| `GET/PUT /api/admin/curriculum-versions` | 130학점 교육과정 버전 관리 |
| `GET/PUT /api/admin/courses` | 과목·학점·규칙 관리 |
| `POST /api/admin/course-demand-surveys` | 설문 생성·공개 |
| `GET /api/admin/course-demand-surveys/:surveyId/summary` | 집계와 익명화된 학년별 수요 조회 |
| `POST /api/admin/annual-offering-plans/:planId/recommendations` | 규칙 기반 초안 계산, 저장하지 않음 |
| `PUT /api/admin/annual-offering-plans/:planId` | 관리자 편집·확정 |

추천 API는 데이터를 변경하지 않는다. 확정 API만 개설 이력에 반영하며, 확정 전마다 서버에서 규칙을 다시 검증한다.

`사진교과교육론`, `사진교수학습방법`의 적합 학기는 초기 과목 마스터 등록 때 관리자가 지정한다. 2년 주기 규칙은 특정 학기 고정 규칙이 아니라, 적합 학기 중 하나에서 개설했는지를 판정한다.

## 9. 오류 처리·감사·개인정보

- 설문 마감 후 학생 응답 변경은 거부한다.
- 학생이 설문 대상 학년이 아니거나 과목이 스냅샷에 없으면 응답을 거부한다.
- 순위는 1부터 연속해야 하며 동일 과목 중복을 거부한다.
- 과목 마스터 변경, 설문 공개·마감, 편성안 확정, 관리자 수동 조정은 감사 로그에 남긴다.
- 관리자 화면은 개인별 원본 응답이 아니라 과목·학년 단위 집계가 기본이다. 개별 응답 열람 권한은 이번 범위에 넣지 않는다.
- 설문 응답의 목적은 학사 편성 지원으로 명시하고, 확정 수강신청 정보나 성적 데이터와 혼용하지 않는다.

## 10. 검증과 완료 기준

### 도메인 테스트

1. 학생은 열린 설문에서만 자신의 대상 학년 후보를 1~5순위로 저장할 수 있다.
2. 중복 과목, 0순위·6순위, 순위 공백, 마감 뒤 수정은 거부된다.
3. 전필이 누락된 편성안은 확정되지 않는다.
4. 사진교과교육론·사진교수학습방법이 최근 2개 학년도에 미개설이면 적합 학기 편성안에서 오류가 발생한다.
5. 일반 과목 운영학점 합계가 85를 넘으면 확정되지 않는다.
6. 캡스톤과 현장실습은 운영학점에 더해지지 않지만 교수인정시수는 계산된다.
7. 현장실습4는 2학기·학생 15·교수 3·운영 0 외의 값으로 확정되지 않는다.

### UI 테스트

1. 학생 대시보드에서 열린 설문 배너와 1~5순위 조작을 확인한다.
2. 모바일에서 순위 재정렬, 저장, 수정, 마감 안내가 접근 가능해야 한다.
3. 관리자 화면에서 85학점 잔여량, 필수 과목 경고, 수요 점수, 학기별 초안을 확인한다.
4. 수동 조정 뒤 오류·경고·사유가 즉시 갱신되어야 한다.

### 완료 기준

- 관리자는 PDF 시드 과목을 검수·등록하고 학년도 편성안을 만들 수 있다.
- 학생은 자신에게 열려 있는 다음 학기 설문에 최대 5개 과목의 순위를 제출할 수 있다.
- 관리자는 수요 집계와 모든 확정된 학점 규칙을 한 화면에서 보고, 규칙을 통과한 편성안만 확정할 수 있다.
- 확정 이력은 다음 편성안의 2년 주기 검증에 사용된다.

## 11. 구현 순서

1. 과목 마스터·학점 규칙·교육과정 버전 데이터 모델과 관리자 등록 화면
2. 학년도 편성안과 85학점·필수·2년 주기 검증 도메인
3. 학생 수요조사 생성·순위 응답·마감 처리
4. 관리자 수요 집계·추천 초안·수동 조정 화면
5. 감사 로그, 접근성, 도메인·브라우저 회귀 테스트

이 순서는 수요조사 응답을 받기 전에 편성 규칙과 과목 마스터가 먼저 신뢰할 수 있어야 한다는 의존성을 반영한다.
